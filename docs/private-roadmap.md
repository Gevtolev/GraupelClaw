# GraupelClaw Roadmap

## Phase 1

- Independent repository and branding
- Centralized project identity in code
- Private-only README and environment setup

## Phase 2

- Fetch OpenClaw `sessions_list` per agent
- Display native session history with `sessions_history`
- Separate local draft conversations from native session history in the UI

## Phase 3

- Visualize agent-to-agent delegation and spawned subagents
- Add filters for session state, origin, and last activity
- Add richer session inspection for tools, artifacts, and yields

## Product Principle

GraupelClaw should become a personal OpenClaw control surface, not just a themed fork of a generic chat product.
